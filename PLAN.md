# 医療系学習アプリ — 実装計画

## 方針の整理

| 項目 | 決定内容 |
|------|----------|
| **プラットフォーム** | まず Web アプリ → iOS は Capacitor でネイティブビルド |
| **コンテンツ** | 資格・試験対策 + 科目別（解剖学・生理学・薬理学など） |
| **継続施策** | ストリーク / デイリーミッション / レベル・XP / ロードマップ / 間隔反復 |

---

## 技術スタック（現状）

| 項目 | 採用技術 |
|------|----------|
| **フレームワーク** | Next.js 15 (App Router) + TypeScript |
| **スタイル** | Tailwind CSS（Duolingo 風の緑基調 UI） |
| **進捗の永続化** | localStorage（将来は Supabase / Firebase 等に差し替え可能） |
| **配布** | Web（静的エクスポート）、PWA、Capacitor による iOS ビルド |

## 実装済みフェーズ

### Phase 1: 土台 ✅
- プロジェクト作成・ルーティング（ホーム / ロードマップ / ブラウズ / レッスン）
- 共通レイアウト（ヘッダー・ナビ）
- 構造化された問題データベース（`src/data/db.ts`）とレッスン単位の出題
- 選択式問題の表示と正誤判定・解説表示

### Phase 2: 継続率アップの 5 要素 ✅
1. **ストリーク** — 最終学習日を保存し、連続学習日数をホームに表示
2. **デイリーミッション** — デフォルト「今日は 5 問」、達成数/目標をホームに表示
3. **レベル・XP** — 正解で +10 XP、100 XP ごとにレベルアップ、ホームに表示
4. **レッスンロードマップ** — 完了レッスンに ✓、前レッスン完了で次をアンロック
5. **間隔反復** — 問題ごとに次復習日を SM-2 風に記録し、レッスン開始時に「今日復習すべき問題」を先頭に混ぜて出題

### Phase 3: コンテンツ・拡張 ✅
- **科目・試験タグでフィルタ** — ブラウズページで科目・試験タグのチップでレッスン一覧を絞り込み
- **問題データの追加・管理** — `src/data/db.ts` / `src/data/tags.ts`、`docs/DATA_FORMAT.md` で形式を明文化
- **分野別ロードマップ・データ管理** — 解剖学・生理学・薬理学・病理学ごとに、難易度順（`orderInSubject`）でレッスンが連なる。ロードマップは分野ごとにセクション表示し、同一分野内で前レッスン完了時に次をアンロック。データは `src/data/subjects/*.ts` で分野ごとに分けて管理し、`db.ts` で集約

### PWA 対応 ✅
- マニフェスト（`app/manifest.ts`）、viewport・appleWebApp メタ、`public/icon-192.png`・`icon-512.png`
- HTTPS 環境で「ホームに追加」「アプリとしてインストール」が可能

### Capacitor（iOS ネイティブビルド）✅
- Next.js の `output: "export"` で静的ビルドを `out/` に出力
- `capacitor.config.ts` で appId `app.medicallearning.web`、webDir `out`
- `ios/` に Xcode プロジェクトを追加、`npm run ios` でビルド → コピー → Xcode 起動

### 問い合わせ・問題報告 ✅
- **お問い合わせフォーム**（`/contact`）— 種別「お問い合わせ」または「問題の報告」を選択可能
- **問題報告** — 報告の種類（問題が誤っている / 誤字脱字がある / その他）、レッスン名・問題ID・問題文を任意で記入。クイズ画面の「この問題を報告」から該当問題の情報を付けた状態でフォームを開ける
- 送信は **mailto** でメールクライアントを起動（送信先は `ContactForm` 内の `MAIL_TO` で変更可能）

### コンテンツ拡充（一部）✅
- 解剖学・生理学・薬理学に問題を追加、**病理学 序論**レッスン（2 問）を追加
